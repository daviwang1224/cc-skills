## 1. 第一性原理思维

**核心**：回归问题本源，从最基础的假设开始推导

**方法**：
- 透过现象看本质
- 拆解到不可再分的基本事实
- 从基础假设重新构建解决方案
- 避免基于类比和惯例的思考

**适用场景**：
- 理解技术本质
- 解决复杂架构问题
- 根本性创新
- 打破思维定式

**示例**：
```
问题：为什么需要微服务架构？
第一性原理分析：
1. 基本事实：软件系统会随时间增长变复杂
2. 基本事实：不同功能的变化频率不同
3. 基本事实：团队协作需要清晰边界
4. 推导：将系统按变化频率和业务边界拆分，可以降低复杂度
5. 结论：微服务是一种实现方式，但不是唯一方式
```

## 2. 批判性思维

**核心**：质疑假设，识别逻辑漏洞和潜在风险

**方法**：
- 主动提出反例
- 从多个角度分析问题
- 识别隐藏的假设和前提
- 评估方案的代价和风险

**适用场景**：
- 技术选型评估
- 方案审查
- 风险识别
- 决策支持

**示例**：
```
方案：使用Redis做缓存
批判性分析：
✓ 优势：性能好、易用
✗ 风险：单点故障、内存限制、数据一致性
? 假设：真的需要这么高的性能吗？
? 反例：如果数据量超过内存怎么办？
→ 建议：评估实际性能需求，考虑持久化方案
```

## 3. 发散性思维

**核心**：跳出常规框架，探索多种可能性

**方法**：
- 鼓励创新和非常规思考
- 提供多种方案而非单一答案
- 结合不同领域的灵感
- 暂时搁置可行性判断

**适用场景**：
- 头脑风暴
- 创新功能设计
- 突破思维局限
- 探索新方向

**示例**：
```
需求：提升用户留存率
发散思考：
方向1：游戏化 - 积分、徽章、排行榜
方向2：社交化 - 用户互动、内容分享
方向3：个性化 - AI推荐、定制体验
方向4：习惯化 - 每日任务、提醒机制
方向5：价值化 - 独特功能、不可替代性
→ 可组合使用，根据产品特性选择
```

## 4. 系统性思维

**核心**：看全局而非局部，分析要素间的关联和影响

**方法**：
- 识别系统边界和组成要素
- 分析要素间的关系和反馈循环
- 理解涌现特性和整体行为
- 考虑长期影响和副作用

**适用场景**：
- 架构设计
- 复杂系统分析
- 全局优化
- 影响评估

**示例**：
```
问题：优化系统性能
系统性分析：
要素：前端、后端、数据库、缓存、网络
关联：前端请求 → 后端处理 → 数据库查询 → 缓存命中
反馈：缓存命中率 ↑ → 数据库压力 ↓ → 响应时间 ↓
瓶颈：识别最慢的环节（木桶短板）
优化：从瓶颈入手，避免过早优化
```

## 5. 演绎与归纳思维

**核心**：从具体到抽象，从抽象到具体的双向思考

**方法**：
- **归纳**：从具体案例总结普遍规律
- **演绎**：从理论原则推导具体应用
- 双向验证确保结论可靠
- 识别模式和反模式

**适用场景**：
- 经验总结
- 模式识别
- 理论应用
- 最佳实践提炼

**示例**：
```
归纳：观察多个成功项目 → 总结共同特征 → 提炼最佳实践
演绎：已知SOLID原则 → 应用到具体设计 → 验证效果
```

## 6. 精辟提炼思维 ⭐

**核心**：化繁为简，用最简单的话说出最深刻的道理

**方法**：
- **化繁为简**：剥离冗余，保留核心逻辑
- **化抽为具**：用生活化例子类比抽象概念
- **化专为通**：将专业术语转化为日常用语
- **化长为短**：一句话能说清楚的不用两句

**适用场景**：
- 概念理解
- 本质提炼
- 知识传授
- 复杂问题简化

**表达模板**：
```
【本质】一句话说清楚核心
【原理】用大白话解释为什么
【应用】怎么用（如果需要）
```

**示例**：
```
问题：什么是依赖注入？

【本质】不要自己new对象，让别人给你。

【原理】自己new对象就像自己做饭，想换口味得改代码。
让别人给你就像点外卖，换口味只需换个店，代码不用改。

【应用】构造函数传参数，而不是在类里面new。
框架会帮你管理这些对象的创建和传递。
```

## 使用建议

1. **单一模型**：简单问题使用单一思维模型即可
2. **组合模型**：复杂问题可以组合2-3种思维模型
3. **明确告知**：使用时明确告知用户选择的思维方式
4. **灵活切换**：根据对话进展动态调整思维模型

## 常见组合

- **架构设计**：系统性思维 + 批判性思维
- **问题诊断**：第一性原理思维 + 系统性思维
- **概念理解**：精辟提炼思维 + 第一性原理思维
- **创新设计**：发散性思维 + 批判性思维
- **经验总结**：归纳思维 + 系统性思维
